Window Maintenance — Astro Static Site (Nextly‑style)

Quick start commands for WSL Ubuntu. This builds a static site for cPanel (no Node runtime required). Analytics are already wired — do not add duplicate tags.

Dev (preview locally)

- Ensure Node 18: `source ~/.nvm/nvm.sh && nvm use 18`
- Install deps: `npm ci`
- Start dev server: `npm run dev` (opens http://localhost:3000)

Build + Package for cPanel

- Build: `npm run build`
- Create deploy zip: `zip -r windowmaintenance-dist.zip dist`
- Deploy: upload the contents of `dist/` to cPanel `public_html` (include `.htaccess`). No server‑side Node needed.

Optional: Copy ZIP to Windows Desktop

- `cp windowmaintenance-dist.zip "/mnt/c/Users/New User/Desktop/windowmaintenance-dist.zip"`

What’s Included

- Tech: Astro + Tailwind; simple, fast, cPanel‑friendly.
- Analytics: GA4 `G-69S1NYW3V6` + Google Ads `AW-16889477810` loaded once in `src/layouts/WMBase.astro`.
- Leads: Inline Formspree form on key pages + mobile sticky Call/Get quote bar. DataLayer event `lead_submit` fired on form submit and the `/thanks` page.
- SEO: Service pages (`/stays`, `/leaks`, `/security`, `/services/*`), areas (`/areas/*`), blog, gallery, canonical tags. Sitemap auto‑generated at build.
- Redirects: Legacy/ad routes mapped in `public/.htaccess` (`/services/leak-repair.html`, `/sliders`, `/windows`, `/hardware`, `/stays`, `/leaks`, `/security`).

Useful Paths

- Analytics + header/footer/layout: `src/layouts/WMBase.astro`
- Global styles + brand/buttons: `src/styles/global.css`
- Home (Nextly‑style): `src/pages/index.astro`
- Services hub + pages: `src/pages/services/index.astro`, `src/pages/services/*`
- Stays/Leaks/Security: `src/pages/stays.astro`, `src/pages/leaks.astro`, `src/pages/security.astro`
- Areas (static‑generated): `src/pages/areas/[area].astro`
- Blog: `src/pages/blog/*`, `src/content/blog/*`
- Forms + sticky CTA: `src/components/QuoteInline.astro`, `src/components/StickyBar.astro`
- Redirects + headers (CSP etc.): `public/.htaccess`

Notes

- Do not modify or duplicate the GA/Ads tags; they are already correct and applied to every page via the layout.
- Formspree endpoint lives in `src/components/QuoteInline.astro` — change there if needed.
- Sitemap is generated by `@astrojs/sitemap` using `site` in `astro.config.mjs`.
